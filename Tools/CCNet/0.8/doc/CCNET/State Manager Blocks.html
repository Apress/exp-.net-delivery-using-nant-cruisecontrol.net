CruiseControl.NET : State Manager Blocks
This page last changed on Oct 17, 2004 by mroberts.

CruiseControl.NET needs to store state about a project. This is data like the last build label, the what the last modifications were, that kind of thing. The State Manager allows you to specify how and where this data is stored.

Versions of CruiseControl.NET prior to 0.7 needed you to specify state manager configuration when you used Multiple Projects on one server. This is no longer the case since the default Project State Manager works with multiple Project configurations.

Types of State Manager Block are:

  • File State Manager
  • Project State Manager
